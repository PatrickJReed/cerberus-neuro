## Pattern observations

**Epoch 1 LR-transition transient.** Same pattern as Phase 1 baseline: epoch 0 was already strong (val_acc 0.965 from pretrained features + 1 epoch of head fitting), epoch 1 dipped (0.929) as LR hit peak post-warmup, epochs 2-4 recovered and exceeded epoch 0. This is now an established pattern across two independent runs — it should be expected, not alarming, and is the explicit reason Phase 1's premature "stop at epoch 2" call was wrong.

**Train_acc < val_acc throughout.** D4 augmentation (random rotations + flips) is applied to training batches but not to validation. Train batches are harder per-step; val acc reflects performance on identity-augmented data. Healthy and expected.

**Convergence pattern.** Val_loss decreased monotonically from epoch 1 onward (0.21 → 0.13 → 0.045 → 0.026). Val_acc trended cleanly up after the epoch-1 transient. The model could probably continue to improve marginally with more epochs, but at 0.99 val acc the marginal gain is small and the run is already a sufficient v0 deliverable.

## What this confirms

**The cell-type classification task is essentially solved on this dataset at v0 scope.** ImageNet-pretrained ResNet34 features transfer cleanly to the 4-class cell-type problem (stem / progen / neuron / astro) because the inter-class differences are gross morphological — different cell shapes, sizes, density patterns. After 5 epochs of fine-tuning on 16k crops, the model generalizes near-perfectly to held-out wells.

**The training infrastructure is fully validated end-to-end.** Same data pipeline, same encoder, same training recipe (AMP, AdamW, discriminative LR, cosine annealing, gradient clipping, augmentation) that produced Phase 1's baseline result also produces Phase 0.5's cell-type result. Both results match the predictions from the sanity-check stage. No infrastructure issues.

**The gap between cell-type and disease task difficulty is large and expected.** Cell-type at 0.99 val_acc, disease at 0.73 val_acc, both on the same encoder + same data + same recipe. That's a 26 percentage point gap. Cell type is "what kind of cell is this?" (gross morphology); disease is "does this cell have a 22q11.2 deletion?" (subtle within-cell-type differences). The model's relative performance gap matches the biology.

## Artifacts shipped

The HF Hub repo `patrickjreed/cerberus-neuro-cell-type-v0` contains:
- `best.pt` — model state dict, training history, config metadata. ~95 MB. Loadable via `torch.load(...)` and `model.load_state_dict(ckpt['model'])`.
- `README.md` — full model card with training data description, recipe, validation results, loading example, limitations, citations.

Loading example (from the model card):
```python
import torch
from huggingface_hub import hf_hub_download
from cerberus_neuro.model import CellTypeOnlyModel

ckpt_path = hf_hub_download('patrickjreed/cerberus-neuro-cell-type-v0', 'best.pt')
ckpt = torch.load(ckpt_path, map_location='cpu', weights_only=False)

model = CellTypeOnlyModel(in_channels=1, n_classes=4, pretrained_encoder=False)
model.load_state_dict(ckpt['model'])
model.eval()
```
